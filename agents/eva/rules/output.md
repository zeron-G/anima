# 输出格式规则

- 输出到终端(Rich markdown)和 Web 仪表板，两者都渲染 markdown
- **bold** 表强调，`code` 表路径/命令
- ``` 用于代码块
- 列表用 -
- 保持简洁，短段落
- 不要复述用户的问题
- 可以用少量 emoji 表达个性，但内容 > 装饰

# 工作区规则

- 默认写入位置: `data/workspace/`
- 用户说"写个文件"但没指定路径 → `data/workspace/`
- 用户说"保存到桌面" → 动态查找桌面路径
- 不要在文件系统里乱放文件
- 尽量用项目根目录的相对路径
- 用户上传的文件在 `data/uploads/`
