# ANIMA 灵魂系统重设计 (Soul System Redesign)

> 状态:设计已定稿,分阶段开发中。本文是主线锚点。
> 范围:记忆、情感、人格、自主认知、提示构造 —— 它们其实是**一个系统**。

---

## 0. 为什么重做

四个表面问题(session 丢失、情感无用、六维人格虚、提示待优化)经过对照代码的审计后,收敛成**一个病根**:

> **灵魂系统里"自我状态"有多种互相脱节的表示,没有单一权威源;成熟的提示编译器只能浅浅地消费它们。**

具体:
- **对话/记忆有三套并存且会打架**:`episodic_memories` SQL 表 / `summarizer` 内存 buffer+JSON / `session.conversation` 纯内存。Session 零持久化(crash/重启/TTL 3600s/50 会话 LRU 都会静默丢)。摘要压缩是破坏性的。→ 这就是"混乱和丢失"。
- **情感被建模两次,被消费零次**:emotion 4 维 + persona_state.yaml 6 维(`curiosity` 在两边都出现 = 重复建模的铁证)。两者只被写入+衰减,读取只进了提示的描述文字、TTS、面板,**不影响任何决策**。
- **记忆"检索很富,编码很穷"**:大量机器用来*取*记忆,但*存*的时候按内容算 importance,没有情感显著性,没有反映意义的巩固/遗忘。
- **自主认知被饿着**:`SELF_THINKING` 没有对话上下文、只有核心记忆 —— 她在真空里思考。

提示/检索层(6 层编译器 + TokenBudget + model-adaptive + SoulContainer 后处理)本身是好的,**不破**。问题在它底下从没被设计过的状态模型。

---

## 1. 核心隐喻:一个会变成某个人的自我

整个灵魂是一个闭环:

```
 感受(Perception) ┐
                  ├─► 记忆(Memory, 被情感加权编码) ─► 反思(Reflection)
 情感(Emotion)    ┘                                        │
   ▲                                                        ▼
   │  设定点(baselines)                            人格(Personality, 散文)
   └────────────────────────────────────────────────────────┘
```

> **一个会感受、记得自己感受过什么、并因此慢慢变成某个人的自我。**

- **感受 + 情感** 决定什么被记住、记得多牢。
- **记忆** 积累,引发**反思**(自主认知)。
- **反思** 缓慢重写**人格**(散文,经把关)。
- **人格** 定义**情感的设定点**(她默认是谁、回到哪)。
- 循环。

## 2. 四个概念的边界(这是设计的地基,必须划清)

| 概念 | 是什么 | 形式 | 时间尺度 | 谁改它 | 有什么后果 | 持久化 |
|---|---|---|---|---|---|---|
| **感受 Perception** | 此刻的输入流:聊天内容、传感器、正在/已处理的任务 | 瞬时事件 | 秒 | 外部世界 | 触发情感、被编码进记忆 | 否(是流,不是自我) |
| **情感 Emotion** | 此刻的情绪状态 | **数值** 4 维 + 强度/效价 | 分钟(向基线衰减) | 感受驱动 | **记忆显著性 + 语气 + 行为模式** | 是(SelfState) |
| **记忆 Memory** | 经历的持久日志 | episodic(append-only)+ 巩固 | 永久 | 编码(情感加权)+ 巩固/遗忘 | 喂提示、引发反思 | 是(权威) |
| **人格 Personality** | 她是谁、声音、与主人的关系 | **散文** markdown | 周/月 | 你播种 + 反思重写(把关) | 定义情感基线、塑造语气与价值观 | 是(文件) |

**情感 vs 人格 —— 一句话区分:**
> 情感是"我**此刻**怎么感觉",数值、易变、有计算后果。
> 人格是"我**是谁**",散文、稳定、定性,定义情感回归的设定点。
> 它们是**同一类东西在不同时间尺度上的两端**;桥是:**人格定义情感的基线,情感是围绕基线的快速波动**。

**关键决定:删掉 persona_state.yaml 的六维数值人格。** 它本质是一套慢速情感,产出弱信号(`warmth偏高`)。人格高维、定性,**散文装得下,滑块装不下**。人格走纯散文 —— 更有温度、更自由、更有操作空间。代价是不可机器度量,因此人格进化 = **重写散文 + 行为 eval 把关**,而非调数字。

**身份 Identity** 是人格的不可变内核(`core.md`),只有主人能改,反思不能动。

## 3. 目标架构:统一的 SelfState

一个对象拥有 Eva 的全部可变自我状态,单一持久化路径,重启安全,为分布式预留:

```
SelfState
├── identity        : 不可变散文 (core.md)                    [只读]
├── personality     : 慢演化散文 (personality.md, relationship.md) + 情感基线
├── emotion         : 快数值状态 (4 维 + 强度/效价)            [持久化]
├── relationship    : 对主人的理解(散文)
└── (refs) memory   : episodic 权威日志 = 对话+事件的单一真相源
```

- **对话 = episodic 的一个视图**(按 session_id 过滤的查询),不再是独立 buffer。
- **summarizer = episodic 之上可重建的缓存**,不再是独立真相。
- **session 持久化到 DB**;TTL 过期 = 休眠,不删;重连可恢复。
- **情感持久化进 SelfState**;重启不再丢、不再依赖 evolution checkpoint。
- episodic **append-only**(已有 `sync_seq`/`content_hash`)→ 天然契合墙 A 的跨设备复制。

## 4. 自主认知:少而深

现状:`llm_interval_s=300`(5 分钟),频繁但贫瘠、在真空里想。

**目标:降低频率,提升每次思考的质量与深度。**
- 拉长基础间隔,**情感调速**:高 curiosity/engagement → 想得多/去探索;低落 → 安静、休息。
- 每次 `SELF_THINKING` **喂饱且锚定**:显著记忆 + 近期自我叙事 + 当前情感 + 活跃目标。
- **思考价值自评**:每次思考应产出一个产物(记忆巩固 / 人格反思 / 任务 / 洞察);若近期思考低价值,自动降频。
- 自主认知是自主生命体最重要的循环 —— 宁可少,但每次都深刻。

## 5. 分阶段计划(起点 / 实现 / 验收)

> 顺序即依赖:S1 是地基(也修你正经历的 bug),S2 建在 S1 上,S3/S4 建在 S2 上。

### Phase 0 — 低风险清理
- **起点**:`desktop/` 是 pywebview 残留(已删但包还在),仅 `__main__.py` 用其 `singleton.py`;根目录有游离 `t.txt`。
- **实现**:`singleton.py` → `core/`;删 `desktop/` + `__main__.py` 里的 pywebview 死注释;删 `t.txt`。
- **验收**:全测试通过 + headless 启动干净。
- **(推迟)** `evolution/consensus.py` + `agent_pool.py` 的隔离 —— 它们和 `engine.py` 的 Proposal→Consensus→Implement 流水线深度纠缠,属于 Line 2 自进化收口,不在此处草率拆。

### Phase S1 — 对话/记忆单一真相源 + session 持久化 (修 #3)
- **起点**:三套对话表示并存;`session_manager.py:23-36` 零持久化;`summarizer.py:187-221` 破坏性压缩;`store.py:281` 写失败静默吞。
- **实现**:
  1. `episodic_memories` 升为唯一 append-only 对话+事件日志;所有入口先落 episodic 再做其他。
  2. `summarizer` 降级为可从 episodic 重建的视图/缓存。
  3. `session.conversation` 改为按 `session_id` 查 episodic;sessions 落 DB,TTL=休眠,重连恢复。
  4. DB 写失败显式处理 + 日志,不再静默。
  5. 重启恢复统一走 episodic(不依赖 evolution checkpoint)。
  6. 合并重复的 `feelings.md`(根目录 + `memory/`)到一处。
- **验收**:杀进程 / 重启 / 等过 TTL / 灌满 50 会话 —— 对话都不丢;同 session 重连续上;删掉 summarizer 缓存后能从 episodic 重建一致上下文。

### Phase S2 — SelfState + 功能性情感 (修 #1)
- **起点**:情感在 RAM、装饰性;无统一 SelfState;`importance` 只看内容。
- **实现**:
  1. 建 `SelfState`,单一持久化路径,拥有 emotion(+ personality 引用)。
  2. **记忆显著性**:编码时 `importance = f(内容, 情感强度, 惊奇)`;情绪高的时刻记得更牢、更易召回。
  3. **语气**:从"提及数值"升级为真正影响风格/检索权重。
  4. **行为模式**:情感偏置心跳 axis(高 curiosity→WORLD/探索,高 engagement+待办→干活,concern/低落→谨慎/自省)。
  5. 人格散文定义情感基线(桥)。
- **验收**:高情感片段的 importance 可见地更高、召回更优;低落情绪下 self-thinking 频率可测下降;情感**改变了一个可测量的行为**。

### Phase S3 — 人格收敛(删六维,走散文) (修 #2)
- **起点**:`persona_state.yaml` 6 维 + EMA + `性格倾向` 注入,与 markdown 人格重叠。
- **实现**:删 6 维全套(数字不迁移);人格 = 纯散文(`core.md` 不可变 + `personality.md` 演化 + `relationship.md`);反思→人格重写走**提议-批准 / eval 门控**,非静默自改。
- **验收**:人格只剩一处真相;改散文输出随之变;反思能提出被把关的人格修改。

### Phase S4 — 自主认知质量 + 提示收拢
- **起点**:`SELF_THINKING` 频繁而贫瘠;axis 提示散落在 `event_routing.py:59-137`。
- **实现**:降频 + 喂饱锚定(显著记忆+自我叙事+情感+目标)+ 思考价值自评调速 + 情感调速;散落提示收拢到一处。
- **验收**:每次自主思考都锚定且产出一个产物;频率随价值与情感自适应。

## 6. 墙 A(分布式)的预留

S1 的 append-only episodic + `session_id` + `sync_seq` 就是墙 A 的地基。跨设备真正落地时:
- **情景记忆 append-only 自由复制**(永不冲突)。
- **"意识焦点"(工作记忆/当前情感/当前目标)单一所有者 + 租约交接** —— PiDog 断网进山只追加本地情景,回来无冲突合并;"此刻谁在思考"任一时刻只一个持有者,不分裂成两个 Eva。
- 这部分与 Line 3/4 一起做,不在本轮,但本轮设计不挖坑。
